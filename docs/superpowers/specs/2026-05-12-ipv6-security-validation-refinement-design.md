## 1. 设计背景

本细化设计将招标 §2.3.1.2「IPv6 安全验证系统要求」中的 6 大功能模块、共 ~25 条要求逐条澄清，结合 Veriguard 现状（§2.4 攻击编排全 + §2.3 B-i / B-iii 完成；B-ii 邮件 inject 进行中；沙箱模块在做），固化每条要求的：

- 业务理解（招标原文 → 平台语义映射）
- 集成边界（具体到甲方既有的哪个系统）
- 默认实现策略（粒度、参数、UI 形态）
- IPv6 适配深度

所有"安全产品"类引用均落到招标 §2.3.1.1「信息化现状」中明确点名的具体系统，避免泛指。

## 4. 模块细化决议

### 4.1 §3 边界防护验证（6 条，含 ★1、★2）

#### 4.1.1 共识

被验证对象：边界设备主要为 **深信服 WAF**、防火墙群（华为 USG6600E / 国基华电 GJSEE-USG / 深信服 AF / 蓝盾 BDFW）。命中告警通过 **蓝盾 NxSOC** 关联。

#### 4.1.2 决议表

| 条 | 决议 |
| --- | --- |
| 3.1 覆盖度验证 + URL 对比 | 资产 = URL 级 web 资产；基线用例 = 用户从用例库中圈定一组；两次对比 = 任意两次同基线 task 间 diff（新增 / 减少 / 状态变化） |
| 3.2 常态化监控 | 监控指标 = "被拦截 / 检出占比"；最小调度粒度 = 小时级 |
| 3.3 稳定性 ★1 | 重复 N 次，**用户可配置，默认 N=10**；指标 = 命中率（命中次数 / N） |
| 3.4 漏洞利用 | 平台内置高危漏洞 PoC 用例库，按软件类别分类（CMS / web 中间件 / OS / DB）；不强制外接 CVE 源 |
| 3.5 WAF/IPS 12 类 + 500 用例 | **双满足承诺**：12 类全覆盖 + ≥500 用例 |
| 3.6 攻击组合 ★2 | 250 攻击类型 × 120 绕过维度 ≈ 30 000 组合；聚类按 (设备, 资产) 维度对未防护项分组；可下钻到 payload 级；平台预置默认分级，支持客户配置 |

### 4.2 §4 流量安全验证（4 条，含 ★3）

#### 4.2.1 共识

被验证对象：**国基华电 GJsec-SIDS、蓝盾 BDIPS（IDS/IPS）+ 奇安信天眼探针**。检出事件统一上送 **蓝盾 NxSOC**，平台从 NxSOC 反查告警。

#### 4.2.2 决议表

| 条 | 决议 |
| --- | --- |
| 4.1 流量覆盖度 | 资产 = IP 级（IPv4 / IPv6 单机）；资产范围按资产组 / 标签筛选；判定链路：平台 → 攻击机 → 目标 → NTA/IDS / 天眼探针 → NxSOC → 平台按 (资产 IP, 时间窗) 查告警 → 提取 `ruleId` → 写入矩阵；矩阵 UI：行=资产 / 列=策略 / **4 态单元格**（✅ 覆盖 / ❌ 未覆盖 / ⏱ 超时未知 / — 不在任务范围） |
| 4.2 设备稳定性 ★3 | 与 §3.3 复用同一机制；N=10 默认；指标=命中率；趋势图按任务一个点（可切按天聚合） |
| 4.3 NTA/IDS 11 类 | **双满足**：11 类全 + ≥300 用例；11 类全部 IPv6 单栈可跑，IPv4 版保留作回归 |
| 4.4 多端口四元组 | 用户可选**顺序 / 并发**（默认顺序），间隔可配；平台**自动改写端口**：单 pcap + 多组四元组声明；仅 dstPort 必须不同，srcPort 系统随机；同协议族（v4 / v6 不混）；单用例最多 64 组 |

### 4.3 §5 应用与服务器安全验证（1 条）

#### 4.3.1 共识

被验证对象：**国家能源集团统建"云眼"**（主机安全 HIDS，全主机部署）。命中告警上送 **蓝盾 NxSOC**。

#### 4.3.2 决议表

| 条 | 决议 |
| --- | --- |
| 5.1 HIDS 12 类 + 300 用例 | **双满足**：12 类全 + ≥300 用例 |
| Agent 形态 | 平台**自有轻量验证 Agent** 与云眼**并存**，提交干扰评估并获甲方批准后安装；不替换云眼、不修改其检测逻辑 |
| 发起方式 | 平台 → 验证 Agent → 主机本地执行（脚本 / 进程 / 文件操作） |
| HIDS 检出判定 | 从 NxSOC 查告警，提取 `ruleId` |
| IPv6 适配 | 网络通信类用例（反弹 shell / 远控 / 隧道 / 暴力破解）必须 IPv6 单栈；纯本地操作类（提权 / 痕迹清理 / 持久化 / 注册表）IPv6 无关 |
| **执行对象硬约束** | **仅在甲方靶机（1:1 仿真主机）上执行；不在生产主机上跑**（与招标 §1.3 靶机方案一致） |
| 自动还原 | 删落盘、回滚注册表、杀进程、清持久化项 |

### 4.4 §6 自定义验证（3 条）

#### 4.4.1 共识

本模块在 Veriguard §2.4 / B-i / B-iii 中已基本实现；本节细化重点是对齐招标语义、统一术语、补齐缺口。

#### 4.4.2 6.1 自定义场景

| 项 | 决议 |
| --- | --- |
| 执行顺序模型 | DAG 编排（复用 Veriguard 攻击链路） |
| ATT&CK 划分 | 用例契约携带 `attack_pattern_id`；矩阵视图 |
| **纵深防御维度（5 层）** | **网络边界 / 流量检测 / 应用层 / 主机端点 / 数据层**；与 §2.1 设备清单 1:1 可观察映射 |
| 维度划分 UI | 场景详情**双 tab**：ATT&CK 矩阵视图 / 纵深防御 5 层视图；用例按其标注自动落到对应格 |
| 批量导入 | Excel 首选 + JSON 可选 |
| 导入策略 | 一次导入 → 一个场景（可拆分）；用例已存在按 hash 幂等 |

#### 4.4.3 6.2 6 类自定义用例

| Veriguard `inject_type` | 招标类型 | 形态 | IPv6 适配 | 现状 |
| --- | --- | --- | --- | --- |
| `http_inject` | 构造 web 攻击包 | HTTP + HTTPS（含自签证书容忍）；method / URL / headers / body / cookies / query | **必须** | ⚠️ GAP，需新增 |
| `pcap_inject` | 上传 pcap 流量包 | 单 pcap + 多组四元组（与 §4.4 承接） | **必须** | ⚠️ B-ii pcap 在 phase 12c 后续 |
| `sample_inject` | 上传样本文件 | **强制沙箱执行**（与 §8 承接） | 无关 | 部分（沙箱在做） |
| `command_inject` | 配置执行命令 | 一行或多行命令；标 `target_os ∈ {linux, windows, both}` | 无关 | ✅ B-i 完成 |
| `binary_inject` | 上传可执行 + 命令 | 二进制 + 启动参数；**靶机执行** | 无关 | ✅ B-i 完成 |
| `email_inject` | 配置邮件形式 | **真实 SMTP**（甲方提供测试邮箱服务器） | **必须** | ⚠️ PR #36 进行中 |

`http_inject` 与 §3.5 平台预置 WAF 攻击用例区分：§3.5 是**平台预置 500+ 完整用例**；`http_inject` 是**用户自定义轻量** PoC。

`binary_inject` 与 `sample_inject` 区分：前者是用户已知用途的工具 / PoC（靶机执行）；后者是可疑 / 真实恶意（沙箱执行）。

#### 4.4.4 6.3 动态用例集联动

| 项 | 决议 |
| --- | --- |
| filter 6 维 | 属性（类型 / target_os）+ 标签 + ATT&CK Tactic/Technique + 杀伤链阶段 + 纵深防御层（5 档）+ 自定义字段 |
| 实时联动 | **任务启动时**实时 evaluate filter（不是创建时一次性 snapshot） |
| 已运行任务 | **不回溯**：保留启动时刻快照，结果稳定可追溯 |
| 场景类型 | **动态 filter 场景 + 手工枚举场景两种共存** |
| 用例库变更提示 | UI 显示"X 条新用例进入场景 Y"；不强制立即重跑 |
| filter 评估窗口 | 任务启动时一次取值；执行中途用例库变化不影响当次任务 |

### 4.5 §7 攻击编排（8 条，含 ★4）

#### 4.5.1 共识

Veriguard §2.4 已全部完成核心能力（编辑画布、运行画布、节点执行配置、SOC 关联、ATT&CK 矩阵视图）。本节细化重点：7.3 分支条件语义、7.4 ★4 执行模式命名、7.5 SOC 匹配维度、7.8 三态结果。

#### 4.5.2 决议表

| 条 | 决议 |
| --- | --- |
| 7.1 可视化编排 + 节点用例自定义 | ChainedTimeline 画布 + 节点 drawer 用例选择（✅ 已实现） |
| 7.2 节点延迟 / 重复 / 间隔 | 延迟单位**秒**（可输秒 / 分 / 时）；重复上限 100；最小间隔 1 秒 |
| 7.3 节点间执行逻辑 | **二元布尔分支**：上一节点被拦 / 未拦；UI 上节点分 on-blocked / on-passed 两条出边；不支持复杂条件 DSL |
| 7.3 "被拦"定义（统一） | 节点期望 expectation = 从蓝盾 NxSOC 收到匹配该节点的告警 → 被拦；超时未收到 → 未拦 |
| 7.4 ★4 验证参数 | 全局参数 + 节点级覆盖（节点级优先级 > 全局）；参数维度：目标资产、时间窗、SOC 对接、超时 |
| 7.4 ★4 执行模式（UI 命名） | **「全程执行」**（无论拦截都继续 / 验证多层防御广度）/ **「拦截即停」**（被拦截后链路终止 / 更贴近真实 APT 行为） |
| 7.5 SOC 关联告警 6 维匹配 | 资产 IP / 时间窗 / 告警严重度 / 规则 ID / 规则类别 / 告警关键字；配置在**全局参数**，节点级可覆盖 |
| 7.6 可视化运行画布 | ChainedTimeline 运行画布（✅ 已实现） |
| 7.7 时间轴展示 | ChainedTimeline 时间轴模式（✅ 已实现） |
| 7.8 链路三态（**防御视角**） | ✅ 已实现（节点级）；⚠️ 链路级三态需新增聚合：**全链路有效**=每节点都被拦/检测；**全链路失效**=每节点都未被拦；**部分失效**=中间状态 |

| 8.2 沙箱平台 CRUD + 网络控制（**截图否则无效**） | CRUD 对象 = 沙箱平台 / 沙箱实例**配置记录**（连接信息 + 网络策略 + 默认参数）；**网络访问控制 4 维**：白名单（出站 IP / 域名）+ 黑名单 + 协议限制（HTTP/HTTPS/DNS）+ 速率限制（出站带宽上限） |

| 还原失败处理 | 标该实例 `unavailable` + 告警通知运维 + 任务自动重排到其他可用沙箱实例 |

## 5. 跨模块约定

### 5.1 IPv6 单栈适配

- 平台 Web 控制台、外部系统接入入口均监听 IPv6 双栈
- 所有"网络通信类"用例（HTTP 攻击、pcap 回放、反弹 shell、远控、隧道、邮件 inject）必须支持 IPv6 单栈目标
- 纯本地操作类用例（提权、痕迹清理、持久化、命令、上传可执行、样本沙箱）与 IPv6 无关
- 与外部组件通信全链路 IPv6 双栈：平台 ↔ 攻击机、平台 ↔ 验证 Agent、平台 ↔ NxSOC、平台 ↔ CAPE 沙箱

### 5.2 靶机执行硬约束（来自招标 §1.3 + §5）

**所有主机侧用例执行（§5、§6.2 binary/command、§7 攻击编排中含主机节点者）仅在甲方"靶机"（1:1 仿真主机）上执行；不在生产主机上跑。** 靶机由甲方在服务器区、办公网区按需部署，与业务无关。生产主机的验证场景需另行协商。

### 5.3 SOC 唯一对接（蓝盾 NxSOC）

- 平台与 SOC 的对接以 **蓝盾 NxSOC** 为**唯一主对象**
- 不直接对接 IDS/IPS / WAF / HIDS / 天眼探针 等检测设备
- 命中判定路径：**目标 → 检测设备 → NxSOC → 平台**（平台从 NxSOC 反查告警）
- 奇安信天眼分析平台保留为备选 SOC（首期不强制）

### 5.4 用例库 + Inject 类型枚举（§6.2 派生）

`inject_type ∈ { http_inject, pcap_inject, sample_inject, command_inject, binary_inject, email_inject }`，作为 Veriguard 用例契约（§7.3）的核心字段。所有自定义用例必须落到其中一类。
