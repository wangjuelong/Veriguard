# IPv6 安全验证系统 — 用例类型与数量清单

> 来源：[`IPv6安全验证系统.md`](./IPv6安全验证系统.md)（招标文件条款汇编）
> 整理日期：2026 年 5 月
> 用途：作为用例库建设、ATT&CK 映射、用例数量验收的基线清单

## 1. 总览

按业务模块汇总，本系统需建设的用例总量与攻击类型最低门槛：

| 模块 | 攻击类型数量门槛 | 总用例数量门槛 | 备注 |
| --- | --- | --- | --- |
| 边界防护验证 — 攻击防护效用 | ≥ 10 种 | ≥ 500 条 | 12 种已具名 |
| 边界防护验证 — 动态组合 | ≥ 250 种 | ≥ 30 000 个组合 | 自动生成；标星 *2 |
| 流量安全验证 | ≥ 10 种 | ≥ 300 条 | 11 种已具名 |
| 应用与服务器（主机）安全验证 | ≥ 10 种 | ≥ 300 条 | 12 种已具名 |
| 自定义验证 — 用例类型 | ≥ 6 种 | — | 用户自创建，无总量门槛 |
| 沙箱管理 — 用例类型 | ≥ 8 种 | — | 真实恶意样本执行 |

**具名用例总量门槛（不含动态组合与自定义）：≥ 500 + ≥ 300 + ≥ 300 = ≥ 1 100 条**

**含动态组合的攻击面规模：≥ 30 000 + 1 100 = ≥ 31 100**

## 2. 边界防护验证用例（≥ 10 种攻击类型 / ≥ 500 条）

适用对象：WAF、IPS 等边界 / 流量防护设备。验证设备能否检出 / 拦截各类边界侧攻击。

### 2.1 已具名攻击类型（12 种）

| 序号 | 攻击类型 | ATT&CK 主要技术域 | 用例配比建议 |
| --- | --- | --- | --- |
| 1 | 命令执行 | TA0002 Execution / T1059 | ≥ 40 |
| 2 | 目录遍历 | TA0007 Discovery / T1083 | ≥ 30 |
| 3 | 弱口令登录 | TA0006 Credential Access / T1110.001 | ≥ 30 |
| 4 | 暴力破解 | TA0006 Credential Access / T1110 | ≥ 40 |
| 5 | SQL 注入 | TA0007 / T1190 | ≥ 60 |
| 6 | 服务器端请求伪造（SSRF） | TA0009 Collection / T1190 | ≥ 40 |
| 7 | 跨站请求伪造（CSRF） | T1659 Content Injection | ≥ 30 |
| 8 | 跨站脚本攻击（XSS） | T1059.007 JavaScript | ≥ 50 |
| 9 | XML 外部实体注入（XXE） | T1190 Exploit Public-Facing App | ≥ 30 |
| 10 | 服务器端模板注入（SSTI） | T1190 / T1059 | ≥ 30 |
| 11 | 畸形表单上传绕过 | T1190 / T1505.003 Web Shell | ≥ 60 |
| 12 | 大包上传绕过 | T1190 / T1499 | ≥ 60 |

**配比合计：≥ 500 条**（满足招标门槛）。

### 2.2 高危漏洞利用及绕过（独立维度）

需覆盖以下产品类目的主流高危漏洞利用 + 绕过手法：

| 类目 | 典型代表 |
| --- | --- |
| 主流 web 服务组件 | Apache / Nginx / Tomcat / IIS |
| 安全产品 | WAF / 防火墙 / 0day 影响面 |
| 应用程序 | 主流 OA / ERP / CMS |
| 国产商业软件 | 用友 / 金蝶 / 通达 / 泛微 / 蓝凌等 |
| CMS | DedeCMS / phpcms / WordPress / Drupal 等 |
| 网络产品 | 路由器 / 交换机 / VPN / 网关 |
| 操作系统 | Windows / Linux / 国产 OS |
| 数据库 | MySQL / PostgreSQL / Oracle / SQL Server / 达梦 / 人大金仓 |

> 此项**不单独计入** ≥ 500 条具名门槛，应与上表中各攻击类型交叉建设——例如"SQL 注入"用例本身就分布在多个数据库的 CVE 上。

### 2.3 动态攻击组合（标星 *2）

| 项 | 门槛 |
| --- | --- |
| 攻击类型可选数量 | ≥ 250 种 |
| 动态生成攻击组合数量 | ≥ 30 000 个 |
| 配套能力 | 设备 / 资产视角自动聚类、不能防护组合的分级 |

> 这是**生成式**指标，不是预编写用例，由攻击组合引擎在 ≥ 250 种基础攻击 × 绕过手法上自动派生；产品需提供功能截图佐证。

### 流量验证的额外约束

> 同一个流量验证用例需支持包含**多个端口不同的四元组**（即一个用例可以涵盖多个 src_ip:src_port → dst_ip:dst_port 流量样本）。

## 5. 自定义验证用例类型（≥ 6 种）

用户**自创建**用例的形式分类（不是预置用例，是平台需要支持的"用例容器"种类）：

| 序号 | 用例类型 | 输入形式 | 执行触发 |
| --- | --- | --- | --- |
| 1 | 构造 web 攻击包 | HTTP request 编辑器（method / url / headers / body） | Agent / 流量回放 |
| 2 | 上传 pcap 流量包 | 上传 .pcap / .pcapng 文件 | 流量回放设备 |
| 3 | 上传样本文件 | 上传二进制 / 脚本样本 | 沙箱 / 主机 Agent |
| 4 | 配置执行的命令 | 命令行文本 | 主机 Agent |
| 5 | 上传可执行文件 + 配置执行命令 | 上传二进制 + 命令模板 | 主机 Agent |
| 6 | 配置邮件形式 | 邮件主题 / 正文 / 附件 / 链接 + 收件人 | 平台 SMTP 直发 |

> 与本仓库二开实施的对应关系：
>
> - PR-B（PR #36）覆盖 #6 邮件形式
> - PR-C 计划覆盖 #1 web 攻击包
> - PR-D 计划覆盖 #2 pcap
> - #3 / #4 / #5 已在 OpenAEV 上游有原生 inject 类型（payload / command / executable），二开继承

### 自定义场景能力

| 能力 | 内容 |
| --- | --- |
| 场景内用例执行顺序 | 用户指定 |
| 场景内用例自动划分 | 基于 ATT&CK / 纵深防御维度 |
| 场景批量导入 | 支持 |
| 场景动态成员匹配 | 用户设筛选条件 → 用例库更新时新增用例自动入场景 |
