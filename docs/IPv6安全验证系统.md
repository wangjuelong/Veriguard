# IPv6 安全验证系统

> 来源：招标文件（国源电力府谷公司 IPV6 单栈试点服务项目）
> 主要条款：§2.3.1.2 IPv6 安全验证系统要求；§2.2 投标人要求（标星条款）；§1.3 项目实施范围与内容；§2.3.6 项目交付
> 整理日期：2026 年 5 月

## 1. 建设目标与适用范围

本系统依托府谷电厂现有 IPv4 安全体系，针对 IPv6 单栈环境下 128 位地址空间特性，构建一套覆盖网络边界、流量侧、主机侧、攻击编排、沙箱等维度的安全有效性验证平台，以"自动化全覆盖、验证闭环"为原则，通过统一的验证任务管理、安全验证场景配置、验证脚本管理为基础，利用攻击验证引擎与多区域部署靶机，进行持续性、可视化的安全有效性验证，实现"攻击未至、防御先行"的主动安全防护。

| 要求内容 | 佐证要求 |
| --- | --- |
| 平台需在 IPv6 单栈环境下提供完整的安全验证能力，针对 128 位地址空间特性重构安全基线 | - |
| 平台与府谷现有 IPv4 安全体系联动，增加对 IPv6 终端的验证特性 | - |
| 平台以"自动化全覆盖、验证闭环"为原则，与安全策略验证措施相结合，形成完整的安全有效性验证框架 | - |
| 平台以统一的验证任务管理、安全验证场景配置、验证脚本管理为基础 | - |
| 平台利用攻击验证引擎和多区域部署靶机方式，开展持续性验证 | - |
| 平台运用可视化技术，实时展现企业安全有效性状态 | - |

## 3. 边界防护验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 对网络边界防护覆盖度进行验证，支持配置覆盖度验证使用的基线用例和对应的策略名称；展示验证的资产总数、防护覆盖资产数、防护未覆盖资产数、站点不同资产数；展示每一个 URL 的防护覆盖情况和具体策略覆盖情况；对比两次覆盖度验证的结果，展示新增资产数、从已覆盖变为未覆盖的资产数、从未覆盖变为已覆盖的资产数、每一个具体 URL 的防护覆盖变化 | 提供功能截图 |
| 对安全设备策略的有效性进行常态化监控，配置监控使用的基线用例和对应的策略名称；通过可视化视图展示每天、每小时的设备有效性常态化监控结果，配置展示设备整体或每个具体策略的监控结果 | - |
| 对边界防护设备和流量防护设备的稳定性进行验证，支持自动通过单次任务重复执行的方式计算一次验证的稳定性结果；支持可视化展示防护设备的稳定性变化趋势 | 需提供功能截图（招标方标星条款 *1） |
| 支持对各类应用和组件的已知高危漏洞利用及绕过的防护有效性进行验证，支持模拟主流 web 服务组件、安全产品、应用程序、国产商业软件、CMS、网络产品、操作系统、数据库的高危漏洞利用特征，验证被测安全防护工具是否能够检出漏洞利用行为 | - |
| 支持验证边界安全设备和策略（如 WAF、IPS），对各类边界侧相关攻击的防护效用，包括：命令执行、目录遍历、弱口令登录、暴力破解、SQL 注入、服务器端请求伪造（SSRF）、跨站请求伪造（CSRF）、跨站脚本攻击（XSS）、XML 外部实体注入（XXE）、服务器端模板注入（SSTI）、畸形表单上传绕过、大包上传绕过；以上类型总用例数量不低于 500 个，或支持的攻击类型不少于 10 种 | - |
| 动态生成攻击手法和绕过方式的攻击组合，以验证边界防护策略，支持选择至少 250 种攻击类型，动态生成至少 30 000 个攻击组合；且能够从设备和资产视角对结果进行自动聚类分析，得出不能防护的攻击组合，并对不能防护的问题进行分级 | 需提供功能截图（招标方标星条款 *2） |

## 4. 流量安全验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 网络边界流量覆盖度验证，可判断网络边界资产是否被流量安全检测设备覆盖，并能够判断每个资产被具体流量安全策略的覆盖情况 | 提供功能截图 |
| 对流量安全设备的稳定性进行验证，可以通过设定任务验证并展示流量安全设备检测稳定性趋势图 | 需提供功能截图（招标方标星条款 *3） |
| 支持验证流量安全设备和策略（如 NTA、IDS），对各类流量侧相关攻击的防护效用，包括：暴力破解、反弹 shell、内存注入 webshell、隐秘隧道、恶意域名解析、webshell 命令执行、高危漏洞利用、远控木马执行、权限绕过、未授权访问、信息泄露；以上类型总用例数量不低于 300 个，或支持的攻击类型不少于 10 种 | - |
| 支持同一个流量安全验证用例中，包含多个端口不同的四元组 | - |

## 6. 自定义验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 自定义场景，能够指定场景内用例的执行顺序，能够基于 ATT&CK 和纵深防御维度对场景内用例进行自动划分；支持通过批量导入用例的形式生成自定义场景 | - |
| 至少 6 种类型的自定义用例，包括：构造 web 攻击包、上传 pcap 流量包、上传样本文件、配置执行的命令、上传可执行文件 + 配置执行命令、配置邮件形式 | - |
| 支持用户从全量用例中进行筛选，将筛选得到的用例创建为自定义验证场景；支持场景包含的具体用例范围根据筛选条件自动变化，当用例更新时，根据筛选条件自动进入自定义场景 | - |

## 7. 攻击编排

| 要求内容 | 佐证要求 |
| --- | --- |
| 支持在可视化页面上，通过自定义路径图的形式配置攻击编排链路，支持自定义配置链路上每个节点的具体用例 | - |
| 支持配置每个节点是否延迟执行、是否重复执行、延迟执行的时间、重复执行的次数和间隔 | 提供功能截图 |
| 支持自定义每个节点之间的执行逻辑，根据上一节点的拦截结果决定是否执行下一节点 | - |
| 使用攻击编排创建任务时，支持全局配置同一套验证参数、每个节点单独配置验证参数；支持配置无论是否拦截都继续执行、被拦截后不再继续执行的两种攻击编排任务的执行模式 | 需提供功能截图（招标方标星条款 *4） |
| 支持使用整个攻击编排链路验证 SOC 关联告警规则，支持配置 SOC 告警规则的匹配条件 | - |
| 支持在可视化页面通过路径图的形式展示攻击编排任务的执行详情，包括攻击编排链路每一个节点的拦截和检测结果 | - |
| 支持通过时间轴的方式展示攻击编排任务中每个节点的执行详情和验证结果 | - |
| 支持区分攻击编排任务的验证结果，包括全链路有效、全链路失效、部分失效 | - |
