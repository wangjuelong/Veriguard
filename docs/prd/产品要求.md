# 2.3.1.2 IPv6 安全验证系统要求

## 一、硬件要求

| 硬件指标 | 指标参数 |
| --- | --- |
| 设备形态 | 2U 机架式 |
| CPU | 国产化 CPU |
| 内存 | 128G |
| 硬盘 | 960G * 2 |
| 网络接口 | 2 端口 10GE 光接口、2 端口 1GE 电接口 |

## 二、功能要求

### 2.1 流量安全验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 网络边界流量覆盖度验证，可判断网络边界资产是否被流量安全检测设备覆盖，并能够判断每个资产被具体流量安全策略的覆盖情况。 | 提供功能截图 |
| 对流量安全设备的稳定性进行验证，可通过设定任务验证并展示流量安全设备检测稳定性趋势图。 | 需提供功能截图 |
| 支持验证流量安全设备和策略，如 NTA、IDS，对各类流量侧相关攻击的防护效用。攻击类型包括：暴力破解、反弹 shell、内存注入 webshell、隐秘隧道、恶意域名解析、webshell 命令执行、高危漏洞利用、远控木马执行、权限绕过、未授权访问、信息泄露。以上类型总用例数量不低于 300 个；或支持的攻击类型不少于 10 种。 | - |
| 支持同一个流量安全验证用例中，包含多个端口不同的四元组。 | - |

### 2.2 应用与服务器安全验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 支持验证主机安全设备和策略，如 HIDS，对各类主机侧相关攻击的防护效用。攻击类型包括：反弹 shell、webshell 上传落盘、命令执行、隧道代理、内存注入 webshell、暴力破解、远控木马执行、系统提权、网站篡改、病毒样本落盘、痕迹清理、主机持久化。以上类型总用例数量不低于 300 个；或支持的攻击类型不少于 10 种。 | - |

### 2.3 自定义验证

| 要求内容 | 佐证要求 |
| --- | --- |
| 支持自定义场景，能够指定场景内用例的执行顺序，并能够基于 ATT&CK 和纵深防御维度对场景内用例进行自动划分。 | - |
| 支持通过批量导入用例的形式生成自定义场景。 | - |
| 支持至少 6 种类型的自定义用例，包括：构造 web 攻击包、上传 pcap 流量包、上传样本文件、配置执行的命令、上传可执行文件并配置执行命令、配置邮件形式。 | - |
| 支持用户从全量用例中进行筛选，并将筛选得到的用例创建为自定义验证场景。 | - |
| 支持场景包含的具体用例范围根据筛选条件自动变化；当用例更新时，符合筛选条件的用例自动进入自定义场景。 | - |

### 2.4 攻击编排

| 要求内容 | 佐证要求 |
| --- | --- |
| 支持在可视化页面上，通过自定义路径图的形式配置攻击编排链路。 | - |
| 支持自定义配置链路上每个节点的具体用例。 | - |
| 支持配置每个节点是否延迟执行、是否重复执行、延迟执行的时间、重复执行的次数和间隔。 | 提供功能截图 |
| 支持自定义每个节点之间的执行逻辑，并可根据上一节点的拦截结果决定是否执行下一节点。 | - |
| 使用攻击编排创建任务时，支持全局配置同一套验证参数，也支持每个节点单独配置验证参数。 | - |
| 支持配置攻击编排任务的执行模式，包括：无论是否拦截都继续执行、被拦截后不再继续执行。 | 需提供功能截图 |
| 支持使用整个攻击编排链路验证 SOC 关联告警规则，并支持配置 SOC 告警规则的匹配条件。 | - |
| 支持在可视化页面通过路径图的形式展示攻击编排任务的执行详情，包括攻击编排链路每一个节点的拦截和检测结果。 | - |
| 支持通过时间轴的方式展示攻击编排任务中每个节点的执行详情和验证结果。 | - |
| 支持区分攻击编排任务的验证结果，包括全链路有效、全链路失效、部分失效。 | - |

### 2.5 沙箱管理

| 要求内容 | 佐证要求 |
| --- | --- |
| 支持使用沙箱环境执行真实恶意样本执行用例。样本类型包括但不限于：勒索病毒样本执行、挖矿病毒样本执行、蠕虫病毒样本执行、终端恶意驱动加载、终端权限提升、终端系统账号窃取、终端代理执行、终端安全组件对抗等。 | - |
| 支持沙箱管理平台的新建、编辑、删除。 | 提供功能截图，否则视为无效 |
| 支持配置沙箱环境的网络访问控制策略。 | 提供功能截图，否则视为无效 |
| 用例执行完成后自动还原沙箱环境。 | 提供功能截图 |
