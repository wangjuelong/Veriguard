# 国内 BAS「真攻击，零伤害」无损模拟技术揭秘

## 文档定位

在网络安全深度融入企业核心业务的背景下，**入侵与攻击模拟（BAS）** 技术面临的核心挑战是：

> 如何在生产环境中执行高度仿真的“攻击”测试，同时确保业务零中断、数据零破坏、系统零不稳定。

国内 BAS 厂商通过一系列无损化技术，解决了这一关键痛点，实现了“在保障业务连续性的前提下，进行高度仿真的实战化攻击模拟”。

无损化模拟技术是 BAS 能够在企业核心生产环境中广泛应用的关键基石，也是其区别于传统渗透测试、红蓝对抗的核心特征之一。

## 核心结论

国内 BAS 的无损模拟能力，主要建立在以下六类技术之上：

1. 流量级无害化与行为可控模拟
2. 探针轻量化与安全自主控制
3. 不对漏洞直接利用
4. 隔离环境与仿真接收端
5. “灭活”样本与威胁控制
6. 内置熔断机制与安全保障

## 技术总览

| 核心技术 | 主要实现方式 | 验证目标 | 无损保障 |
| --- | --- | --- | --- |
| 流量级无害化 | 数据包标记、流量重放、动态 ID | 验证 WAF、IPS 等安全设备的检测能力 | 不触发真实漏洞，不修改系统状态 |
| 轻量化探针 | Agent 热加载、休眠、任务后卸载 | 模拟本地攻击行为与终端侧检测 | 降低资源占用，减少驻留风险 |
| 不直接利用漏洞 | 发送漏洞特征请求而非真实 Payload | 判断防护设备是否识别攻击意图 | 只验证暴露面，不真实入侵系统 |
| 隔离与仿真接收端 | 沙箱、虚拟网络、快照回滚 | 模拟高风险样本或零日攻击 | 与生产环境隔离，可快速恢复 |
| “灭活”样本 | 保留行为特征，移除破坏代码 | 验证病毒、木马、勒索软件检测能力 | 不具备真实感染和破坏能力 |
| 内置熔断 | 监控 CPU、内存、流量、延迟等指标 | 控制模拟攻击的运行边界 | 异常时自动终止并回滚 |

## 一、流量级无害化与行为可控模拟

通过**数据包标记（Packet Tagging）** 与**流量重放**技术，在不改变网络拓扑、不部署额外攻击设备的前提下，精确控制和追踪模拟攻击流量。

### 核心原理

在生成的模拟攻击数据包中嵌入特殊且唯一的标识符，也就是流量指纹。BAS 平台通过这些标识符区分模拟攻击流量与真实攻击流量，并跟踪安全设备的检测、告警和阻断结果。

### 应用示例

- **模拟 SQL 注入**：不发送真正破坏数据库的恶意代码，而是在查询语句中嵌入无害的特殊标记，例如 `--BAS_TRACE_XYZ`。该流量经过 WAF、IPS 等安全设备时，会触发检测逻辑，BAS 通过捕获设备告警或阻断信息来验证防御能力。
- **模拟端口扫描**：发送带标记的探测报文，并严格限制连接数量和频率，避免被误判为 DDoS 攻击或导致服务过载。

### 核心价值

在不实际触发漏洞利用、不真实修改系统状态的前提下，精准验证安全设备的检测能力。

### 厂商案例

- **长亭科技**：通过对业务的仿真能力，模拟各种业务环境的响应，无需部署真实靶机。
- **绿盟科技**：采用流量标记技术，在模拟攻击时将载荷自动替换为测试语句。
- **墨云科技**：采用流量重放和插入动态 ID 技术，区分模拟流量与真实流量。

## 二、探针轻量化与安全自主控制

通过在目标服务器部署轻量级 Agent（探针）来模拟本地攻击行为，并确保其高度安全、可控。

### 攻击模块热加载

- Agent 平时仅为加载了基础通信模块的“空壳”程序。
- 仅在执行特定安全验证任务时，临时动态加载所需的攻击或仿真模块。
- 任务完成后立即卸载模块，实现“用完即走”的无痕验证，降低 Agent 自身的安全隐患和资源占用。

### Agent 安全模拟实例

- **模拟恶意进程**：不在真实系统启动 `cmd.exe`，而是在系统日志中模拟进程创建事件，或在受控虚拟环境中模拟运行，以观察 EDR 告警。
- **模拟文件操作**：创建虚拟文件或在内存中进行操作，不实际写入磁盘。
- **模拟数据外传**：仅发送空包或带特殊标记的无害数据，避免真实敏感数据泄露。

### 厂商案例

- **华云安**：采用攻击模块热加载技术，实现探针的临时加载与卸载。
- **塞讯科技**：采用 Agent 休眠技术，无任务时休眠，有任务时被唤醒。
- **矢安科技**：采用热加载技术，任务完成后自动还原清理执行痕迹。

## 三、不对漏洞直接利用

这是 BAS 与传统自动化渗透测试的核心区别。BAS 仅模拟攻击者尝试利用漏洞的行为，但不会实际触发漏洞并侵入系统或修改数据。

### 工作原理

发送仅符合漏洞触发特征、但不包含真实恶意 Payload 的请求，观察安全设备，例如 WAF、IPS，是否能够检测并阻断。

### 示例：远程代码执行漏洞验证

1. BAS 发送不含真实恶意命令的漏洞特征请求。
2. 观察 WAF、IPS 是否检测并阻断。
3. 如果设备未检测到，BAS 仅记录“该漏洞可被利用”，但**不会**发送真实的攻击 Payload。

### 核心价值

实现“只探虚实，不伤筋骨”的无损伤风险评估，避免在生产环境中制造新的安全风险。

### 厂商案例

- **灰度安全**：通过特定的攻击向量、评估机制和算法进行无害化风险评估。
- **塞讯科技**：仅发送触发特征的请求，验证对外暴露业务的防护能力。
- **矢安科技**：技术手段使攻击模拟时仅让安全设备做出回应，后端目标系统不真实响应。

## 四、隔离环境与仿真接收端

隔离环境与仿真接收端可理解为用于模拟高风险攻击行为的“安全气泡”，常用于零日漏洞、勒索病毒等场景。

### 技术实现

在高度受控的虚拟化环境，也就是沙箱中运行真实的恶意样本，测试安全设备的检测能力。

### 安全保障

- **网络隔离**：拥有独立的虚拟网络和内置 DNS 服务器，响应虚假 IP，阻止恶意软件真实外联。
- **环境回滚**：测试完成后，通过**快照技术**快速将环境回滚到初始纯净状态。

### 应用示例：勒索病毒攻击模拟

- 在隔离沙箱中投放勒索样本，观察其尝试加密虚拟文件的行为，以及终端安全产品的反应。
- 整个过程完全不影响生产网络中的实际服务器和文件。

### 厂商案例

- **绿盟科技**：采用沙箱隔离技术，实现零侵入攻击模拟。
- **塞讯科技**：采用“受保护的沙盘”，具备独立虚拟网络和快照回滚机制。

## 五、“灭活”样本与威胁控制

通过对真实恶意样本进行无害化处理，使其从“毒药”变为安全的“试剂”。

### 灭活处理技术

对真实恶意样本进行分析和重写，在**保留触发安全检测的关键行为特征**，例如文件扫描、提权尝试的同时，**彻底移除其核心有害代码和破坏性操作**，例如文件加密、删除、建立真实通信隧道。

### 示例：勒索软件灭活

| 行为特征 | 真实勒索软件 | 灭活勒索软件样本 |
| --- | --- | --- |
| 文件读取 | 保留 | 保留 |
| 文件重命名 | 保留 | 保留 |
| 生成勒索信息弹窗 | 保留 | 保留 |
| 文件加密模块 | 存在 | 移除 |

### 核心价值

能够安全地测试防御系统对病毒、木马、勒索软件等的检测和阻断能力，而无需承担感染风险。

### 厂商案例

- **塞讯科技**：提供无害化样本或模拟恶意样本行为进行验证。
- **矢安科技**：通过重写攻击样本，保留特征、去除危害，得到“灭活的病毒”样本用于各类验证场景。

## 六、内置熔断机制与安全保障

内置熔断机制是模拟攻击过程中的“安全保险丝”，用于在出现意外风险时自动干预，确保测试过程可控。

### 监控指标

系统持续监控目标系统的关键运行指标，包括：

- CPU 使用率
- 内存占用
- 网络流量
- 服务响应延迟
- 关键文件变更
- 异常网络模式

### 熔断机制

一旦检测到任何异常，或系统指标超出预设安全阈值，BAS 平台会**立即自动终止测试任务**。

### 配套安全原则

- **最小权限原则**：BAS 平台仅获取完成任务所需的最小权限。
- **环境回滚机制**：确保发生意外时能快速恢复到验证前的安全状态。

### 厂商案例

- **绿盟科技**：产品内置熔断机制，当触发系统异常，例如 CPU 激增时，自动终止测试并回滚状态。

## 总结

通过**流量级无害化、探针轻量化控制、不直接利用漏洞、隔离环境、样本灭活、内置熔断**六大核心技术的综合运用与创新，国内 BAS 厂商构建了多层次、纵深化的安全保障体系，使“真攻击，零伤害”的无损模拟成为现实。

这让企业能够放心地在核心生产区域开展持续、实战化的安全防御能力验证，并将安全投入转化为可衡量的防御能力和业务价值。
