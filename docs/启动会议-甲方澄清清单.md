# 一、紧急澄清事项（启动第 1 周内必答）

## 1.1 安全运营平台对外接口规范

**关注点**：网络边界覆盖度、流量安全验证、主机安全验证、攻击编排链路四大核心场景，最终都需要从甲方安全运营平台读取关联告警以计算「是否被检测到」。接口若不可用，验证结果中的「检测维度」判定整套失效。

**待澄清**：

- 接口形态：REST API / gRPC / Kafka 订阅 / syslog 转发？
- 认证方式：API Key / OAuth 2.0 / mTLS / 无认证？
- 六维查询字段映射：资产 IP、时间窗、严重度、规则 ID、规则类别、关键字 六维 ↔ 平台实际字段名
- 是否同时支持 IPv6 和 IPv4 双栈调用

**项目方倾向答案**：REST API + Bearer Token + 双栈支持。

**备选方案**：若不开放查询 API，改为 syslog 转发 + 本平台自建中转网关 + Elasticsearch 落盘 + 自实现查询。增加约 2 周工作量但不阻塞主线。

**风险**：第三方厂商技术支持响应周期可能拖至 2-4 周。建议第 1 周内甲方提需求，项目方同步启动备选方案。

---

## 1.2 沙箱系统对接

**关注点**：真实恶意样本（勒索病毒 / 挖矿 / 蠕虫 / 木马驱动等八类）执行模块依赖项目方提供的沙箱能力。沙箱模块需支持八类样本执行、用例完成后自动还原、网络访问策略下发。

**待澄清**：

- 单节点并发样本数上限
- 八类样本各自的样本提供渠道

**项目方倾向答案**：

- 沙箱由项目方整体承包，对甲方为黑盒能力（不暴露底层实现细节）
- 网络访问策略只提供「联网 / 断网」两种最简状态
- 八类样本为硬性交付要求，每类至少入库 5 个家族变种
- 用例完成后自动还原走快照恢复路径；异常场景（还原失败 / 沙箱主机宕机）走人工介入流程，不做自动兜底

**风险**：沙箱评审 + 版本号锁定需在第 1 周内完成，避免后续模块对接受阻。

---

## 1.3 靶机部署细则

**关注点**：靶机是项目实施的物理基础，其数量、操作系统、网络配置由甲方根据实际业务场景确定。

**待澄清**：

- 靶机总数与操作系统清单（由甲方按业务系统拟定）

**项目方倾向答案**：靶机统一配置 IPv6 + IPv4 双栈；具体数量与操作系统类型由甲方提供。

**风险**：靶机交付时限会影响主机侧验证模块的落地节奏，建议第 1 周内甲方提供初步清单。

---

## 1.4 集团 IPv6 资产管理平台对接

**关注点**：招标要求与集团 IPv6 资产清单联动。若资产清单不准确，则验证目标对不上，导致网络边界覆盖度验证无法完整执行。

**待澄清**：

- 集团资产管理平台的对外接口（REST / Kafka / 文件同步）
- 数据上报格式与频率
- 认证方式

**项目方倾向答案**：REST API 主选；若不可得，每日定时拉取 Excel / CSV 文件。

**风险**：集团 IT 跨部门协调可能需要 2-3 周。

# 二、重要澄清事项（启动后 2-4 周补齐）

## 集成对接类

### 2.1 安全设备清单与策略导出格式

**关注点**：覆盖度验证需要把「用例 × 策略 × 资产」做矩阵展示。若策略名称对不上，则矩阵为空。

**待澄清**：

- WAF / 防火墙 / IPS / IDS / HIDS 已部署设备清单（厂商、型号、版本）
- 各设备策略导出格式（厂商 API / 配置文件 / Excel 报表）
- 策略命名规范

**项目方倾向答案**：API 优先，无 API 时走 Excel 模板 + 定期同步。

---

## 用例数量与内容类

### 2.2 边界攻击用例 12 类配比

**现状**：项目方已超额完成边界攻击用例库建设，12 类全集已就绪，远超招标 ≥ 500 条门槛。

**待澄清**：验收阶段的抽样策略（每类抽样数量 + 抽样方式）。

---

### 2.3 三万攻击组合执行窗口

**现状**：三万攻击组合是平台的工程能力指标（已具备动态生成 + 聚类 + 分级能力），不要求单次任务必须跑满 3 万。

**待澄清**：

- 验收阶段单次任务实际执行规模（全量 / 抽样数百 / 仅看能力演示）
- 演练流量标识机制：每条演练流量带专有标识（HTTP header / packet TTL），便于甲方运营平台过滤，避免误判为真实攻击

---

### 2.4 流量用例 IPv6 单栈覆盖度

**现状**：NTA 流量用例 11 类已超额完成，IPv6 单栈为主。

**待澄清**：IPv4 兼容版本是否需要补充（按客户业务实际需求）。

---

## 验收判定规则类

### 2.5 攻击编排 SOC 六维匹配字段

**关注点**：攻击编排链路与 SOC 告警关联匹配，是标星条款强制项。

**待澄清**：

- 六维（资产 IP / 时间窗 / 严重度 / 规则 ID / 规则类别 / 关键字）↔ 安全运营平台实际字段名映射表
- 是否允许节点级覆盖全局参数
- 规则 ID 是平台 ID 还是设备 ID

**项目方倾向答案**：六维通用模型 + 安全运营平台适配层做字段翻译。

**依赖**：本项依赖紧急澄清 1.1（安全运营平台接口）完成后才能定。

---

### 2.6 沙箱网络访问控制四维策略

**关注点**：招标「沙箱平台网络访问控制策略」为强制截图项，否则视为无效。

**待澄清**：

| 维度 | 内容 |
| --- | --- |
| 白名单 | 允许出网的 IP / 域名 |
| 黑名单 | 禁止访问的 IP / 域名 |
| 协议 | HTTP / DNS / ICMP 各自允许 |
| 速率 | 单连接带宽 / 总带宽限制 |
| 预设策略 | 完全隔离 / 仅 DNS / 仅外网 / 受控外网 |

**项目方倾向答案**：四维完整 + 4 个预设策略，由沙箱适配层下发。

# 三、一般澄清事项（试运行期补齐）

> 以下项不阻塞主线研发，可在试运行期通过 weekly review 补齐。

### 3.1 用户角色与权限模型

四个默认角色（管理员 / 操作员 / 审计员 / 只读）已实现 RBAC 权限矩阵。若客户要求集成集团 IAM 角色映射，在试运行期补适配。

### 3.2 平台可用性与容灾

默认承诺 99.9% SLA + 单机部署。如需 HA 主备，追加硬件后实施。PostgreSQL 每日全备 + Elasticsearch snapshot；RTO 4 小时 / RPO 24 小时默认。

### 3.3 升级维护窗口

默认每月第 1 周末 22:00-02:00 维护窗口。升级走容器滚动重启 + 数据库迁移；DB 备份 + 镜像版本锁定支持回滚。

### 3.4 数据导入导出格式

默认 Excel 模板（用例批量导入）+ JSON 导出（链路 / 配置）。支持 STIX 2.1 + ATT&CK 标准格式，与 MITRE 生态互通。

### 3.5 报告输出频率与渠道

默认 dashboard 实时 + 月报 Word + 季报 Word + 突发事件即时报告。邮件订阅每月 1 号自动发送。

### 3.6 性能基线承诺

| 指标 | 默认承诺 |
| --- | --- |
| 平台并发任务数 | ≥ 100 |
| 三万攻击组合执行时间 | ≤ 4 小时（限流 50 并发） |
| 单次 inject 的 P95 延迟 | ≤ 5 秒 |
| Agent 心跳间隔 | 30 秒 |
| Agent 结果上报间隔 | 1 分钟 |

### 3.7 IPv6 单栈程度

默认平台内部纯 IPv6 + 外部对接（安全运营平台 / 沙箱 / SMTP / 集团 IPv6 平台）双栈。DNS / NTP / 镜像源用 IPv6 链路。

### 3.8 培训教材的产权与延续

教材项目方主编 + 甲方共享版权（甲方可二次分发）。培训后 1 年内 ≤ 2 次复训免费。线上培训含录像存档。

### 3.9 攻击特征库更新机制

默认每月 1 次特征库更新（用例库 / 漏洞库 / 攻击样本库同步）+ 紧急 CVE 24 小时内补充。离线更新方案：项目方提供升级包，甲方人工导入。

### 3.10 多区域部署网络拓扑细节

默认每个「区域」独立 VLAN + IPv6 段。跨区域流量经客户既有防火墙。具体 IP 段在靶机部署细则中一并对齐。
